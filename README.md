###日麻數據整合與技術趨勢追蹤系統

## Proposal Report

### 動機與目標
<!-- 說明為什麼想做這個專題 -->
目前在進行日本麻將技術檢討時，我主要依賴以下兩大工具：

雀魂牌譜屋：提供大量的長期統計數據（如和了率、放銃率等），側重於「結果」的呈現。例如：即使打錯牌（惡手）但最終和牌，在牌譜屋的數據紀錄中仍會被視為正向結果。

Mortal AI：提供單場比賽中每一手捨牌的技術分析與評分 (Rating)，側重於「過程」與決策品質的檢討。例如：即使選擇了理論最優解（正解）但最終放銃，在 Mortal 的數據中仍會給予高評價。

然而，在實際應用中發現，這兩者之間存在一個分析斷層：牌譜屋的數據過於宏觀，難以察覺短期的技術波動；而 Mortal 的分析過於微觀，難以整合出長期的「惡手傾向」或「技術進步曲線」。因此我想結合兩者開發一個系統，獲取數據後提供雀士一個綜合評價。

### 預期功能
<!-- 列出預計實作的功能 -->
本系統將針對錄入數據進行多維度分析，產出以下三大核心評價：

(1) 玩家風格綜合評價
分析邏輯：利用「副露率」、「立直率」與「平均打點」...等進行分析。

輸出標籤：系統將自動標註玩家風格，例如：

「門前進攻型」 (高立直率、低副露、高打點)

「副露速度型」 (高副露、低巡數和了、低打點)

「防守型」 (低放銃率)

(2) 運氣偏離值分析 (Luck/Variance Analytics)
分析邏輯：透過交叉比對 Mortal Rating 、 Mortal 惡手率(過程品質) 與 終局順位 (結果表現)。

輸出結果：

「運氣欠佳」：Mortal Rating 極高但順位多為 3、4 位。

「運氣優勢」：Mortal Rating 偏低但順位多為 1、2 位。

這能幫助玩家區分哪些失敗是「技術問題」，哪些是「純粹運氣」，避免情緒波動影響判斷。

(3) 技術水平追蹤 (Skill Level Tracking)

分析數據每十場做一次技術總結，提供圖表直觀的觀察。

### 使用技術
<!-- 使用的語言、框架、工具等 -->
本專案預計使用 Python 作為核心開發語言，並導入以下技術模組：

資料格式 (Data Storage)：使用 JSON 格式。

原因：JSON 與 Python 的字典 (Dictionary) 結構完美契合，適合儲存戰績這種多層次的資料，且易於手動維護與擴充。

數據處理 (Data Processing)：使用 Pandas 函式庫。

原因：這是資料科學的標準工具，可以輕鬆計算「每十場的移動平均」以及各種複雜的統計指標。

數據視覺化 (Visualization)：使用 Matplotlib 與 Seaborn。

原因：能產出專業的走勢圖、直方圖與雷達圖，讓「技術成長」與「運氣偏差」一眼就能看出來。

### 與課程的關聯
<!-- 你的專題可能涉及哪些資料結構或演算法概念？為什麼？ -->
數據建模與物件導向 (OOP & Data Encapsulation)
課程關聯：Class, Objects, Encapsulation。

實作說明：每場比賽不只是單純的數字，而是一個具有多種屬性（Score, Rank, Rating, Akuma-rate）的 Object。我會定義 GameRecord 類別，將「數據（Data）」與「評價邏輯（Methods）」封裝在一起。這能避免程式碼變成混亂的二維陣列，並確保未來在擴充新指標時，符合模組化設計的原則。
